# UVM
> Uni-Virtual Machine - Atomic orchestration enables maximum flexibility

Uni-Virtual Machine (UVM) is a production-grade, atomic multi-agent orchestration engine. It provides a secure, stack-based bytecode VM, unified memory management, and a domain-specific language (USL) for composing and executing complex agent workflows — including **AI-generated dynamic workflows that can be compiled and executed at runtime**.

## Why UVM?

Traditional agent frameworks use static, pre-defined workflows. UVM enables **dynamic workflow generation**: an LLM can write a USL program describing a multi-step agent collaboration, and UVM compiles and executes it immediately — safely, without using `exec()` or Python sandboxing.

This unlocks truly adaptive multi-agent systems where the orchestration strategy itself is generated by AI based on the problem context.

## Features

- **Dynamic Workflow Execution**: AI-generated USL code is parsed, compiled to bytecode, and executed in real-time on a secure VM
- **Atomic ISA**: Procedure-oriented instructions (CALL, AGENT, EXEC, etc.) for composing multi-agent workflows
- **USL (UVM Specific Language)**: A lightweight DSL with loops, conditionals, and function calls for agent orchestration
- **Stack-based Bytecode VM**: Memory-optimized execution with yield/resume for checkpointing and hibernation
- **Unified Memory Management**: `MemStr` with lifecycle hooks for external memory engines (KV-cache eviction)
- **Pluggable Memory Engine**: Interface for custom memory managers to inspect agent structure and CALL events
- **Parallelism-Friendly**: Thread-safe `UVMContext` using `contextvars` for isolation
- **OpenAI Integration**: Native support for OpenAI API with per-LM configuration

## Quick Start

### Installation

```bash
pip install pyyaml openai
```

### Basic Usage

```python
import uvm
from uvm.ISA import *

# Create LMs with configuration
m1 = uvm.LM(config="gpt4_config.yml", name='m1')
m2 = uvm.LM(config="claude_config.yml", name='m2')

# Call directly or via CALL instruction
response = m1("Hello, let's start!")
response = CALL("m1", "Hello via ISA!")

# Create agents with harness
AGENT('planner', 'm1', 'You are a planning agent')
AGENT('coder', 'm2', 'You are a coding agent')

# Execute USL workflow
plan = '''
step = "analyze"
result = CALL("planner", "Plan this task:", step)
RETURN(result)
'''
result = EXEC(plan)
```

### Dynamic Workflow Example

UVM's killer feature: LLM-generated workflows executed at runtime.

```python
# Create specialized agents
AGENT('judger', 'm1', 'You decide which tool to use: [search, calculate, code]')
AGENT('executor', 'm2', 'You execute the chosen tool')
AGENT('checker', 'm1', 'You check if the task is complete. Reply YES or NO')

# Get a problem from user
problem = INPUT()

# Ask the planner to generate a USL workflow
planner_prompt = f"""
Write a USL workflow to solve this problem: {problem}

Available agents: judger, executor, checker
USL syntax:
  - variable = CALL("agent", "prompt", ...)
  - DO(body)WHILE(condition)
  - IF(condition, {{then_body}}, {{else_body}})
  - RETURN(value)

Example:
step = "initial data"
DO(
    tool = CALL("judger", "Which tool for:", step)
    step = CALL("executor", "Execute", tool, step)
)WHILE(CALL("checker", "Is task done?", step) == "NO")
RETURN(step)
"""

# The LLM generates USL code
generated_usl = CALL("planner", planner_prompt)
print(f"Generated workflow:\n{generated_usl}")

# Execute the AI-generated workflow immediately
result = EXEC(generated_usl, initial_vars={"problem": problem})
print(f"Result: {result}")
```

This enables truly adaptive multi-agent systems where the orchestration strategy is generated on-the-fly by AI based on the problem context.

## Architecture

### Core Components

| Component | Description |
|-----------|-------------|
| `UVMContext` | Thread-safe execution context holding LM/Agent registries |
| `MemStr` | Memory-managed string with lifecycle hooks |
| `LM` | OpenAI-backed language model wrapper |
| `AGENT` | LM wrapper with harness for memory control |
| `HARNESS` | Input/output transformation hooks |
| `MemoryEngine` | Pluggable interface for external memory management |

### USL Bytecode VM

USL code is parsed, compiled to bytecode, and executed on a stack-based VM:

```
USL Source → Parser → AST → Compiler → Bytecode → UVM (execution)
```

The VM yields at every `CALL` instruction, allowing the runtime to:
- Check memory pressure
- Checkpoint and hibernate
- Page out cold agent contexts

### Memory Engine API

Third-party memory engines implement:

```python
class MemoryEngine(ABC):
    def on_agent_register(self, agent_struct: AgentStruct) -> None
    def on_agent_unregister(self, name: str) -> None
    def on_call_enter(self, event: CallEvent) -> Optional[Any]
    def on_call_exit(self, event: CallEvent, handle, result: MemStr) -> None
    def on_memstr_create(self, ms: MemStr, context: str) -> None
    def on_memstr_drop(self, ms: MemStr) -> None
    def checkpoint_request(self) -> bool
```

Included implementations:
- `NullMemoryEngine`: No-op (default)
- `LoggingMemoryEngine`: Debug logging
- `CountingMemoryEngine`: Statistics tracking

## ISA Reference

### Core Instructions

| Instruction | Description |
|-------------|-------------|
| `CALL(name, *args)` | Invoke LM or Agent by name |
| `AGENT(name, lm, prompt)` | Create and register an agent |
| `INPUT()` | Read user input |
| `RETURN(value)` | Return from USL execution |
| `EXEC(plan)` | Execute USL code |

### Control Flow (USL)

```python
# Variable assignment
x = "hello"

# Binary operations
y = x + " world"
result = x == "hello"  # True

# DO-WHILE loop
DO(
    step = CALL("agent", "process", step)
)WHILE(CALL("checker", "should_continue") == "YES")

# IF statement
IF(condition == "true", {
    result = CALL("agent_a", "task")
}, {
    result = CALL("agent_b", "task")
})

# Nested EXEC (dynamic workflow execution)
sub_plan = CALL("planner", "Generate sub-workflow")
result = EXEC(sub_plan)  # Execute generated USL at runtime

# BREAK
BREAK()  # Exit current loop
```

## Parallelism

UVM uses `contextvars` for thread-safe context isolation:

```python
import uvm
from uvm.core import UVMContext, set_context

def worker(task):
    ctx = UVMContext()
    set_context(ctx)
    # ... use UVM in this thread
    return CALL("agent", task)

# Each thread has isolated context
with ThreadPoolExecutor() as pool:
    results = pool.map(worker, tasks)
```
